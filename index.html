MediBot — Compact Pi 3 A+ Design

A compact, practical robot that recognizes labelled medicines, extends an arm to pick them, and delivers them on schedule. Single‑board computer (Raspberry Pi 3 A+), Pi Camera, wheeled base and an articulated arm.

Overview

This design targets a finished, product-like look while keeping component costs low. MediBot stores a small medicine database, uses on-device image processing (Tesseract / Pyzbar / EasyOCR) on the Pi to read labels or QR stickers, and runs a scheduled dispense routine using a DS3231 RTC or the Pi's clock. The arm uses hobby servos for pick/reach and small DC motors for mobility.

Use cases

  • Home medication reminders and automated retrieval at scheduled times.
  • Assisted living: bring prescribed bottles to a fixed drop point.
  • Prototype for pharmacy‑automation demos.

Mechanism of functioning

  1. Pi Camera scans shelf/label when MediBot is at the storage position or when user requests a scan.
  2. Pi runs a small image pipeline: preprocess → QR/Barcode decode or OCR → lookup ID in local DB (JSON/SQLite).
  3. On a schedule match, motion controller executes: rotate base toward slot (differential drive), position arm using taught joint angles, close gripper, retract and move to drop point, release.
  4. Events are logged to local storage (microSD) and optional Wi‑Fi notification (Telegram/HTTP webhook).

Key software components

On‑device
  • Python 3 + OpenCV for capture & preprocessing
  • pyzbar or zxing for QR/barcode detection
  • Tesseract or EasyOCR for printed labels (optional)
  • RPi.GPIO / pigpio or gpiozero for motor & servo control
  • SQLite or JSON for local medicine DB
Optional cloud/phone
  • Flask server / ngrok for remote decode (if needed)
  • Telegram bot or local web UI for manual control

Safety & constraints

Keep carried weight small (under 200 g recommended). Use proper power protection (BMS, fuses) and avoid contact between the gripper and medication surfaces when hygiene is required — use small smart cups or liners.

Build time & step-by-step procedure

Estimated hands-on time: 4–8 hours (spread across cutting/printing, wiring, software, testing). If you are doing your first battery pack and wiring, allow extra time for safety checks.

Step 0 — Safety & prep (30–45 minutes)

  1. Verify you have all parts and safety gear (multimeter, small screwdriver set, heatshrink, soldering iron if needed).
  2. Prepare a non‑flammable workspace for battery assembly and charging. Read BMS and charger instructions.

Step 1 — Mechanical base & arm (1.5–2 hrs)

  1. Cut the MDF chassis plates (base, uprights) using a template. Drill mounting holes for motors and Pi. Sand edges for a finished look.
  2. Assemble wheel motors to base, mount caster if needed. Attach vertical stand for medicine shelf.
  3. 3‑link arm: mount servos into printed/laser‑cut brackets, use bolts as hinge pins. Attach a two-finger gripper to the last servo.

Step 2 — Electronics & power (1–2 hrs)

  1. Assemble 2S3P pack and securely solder or spot‑weld nickel tabs, connect to 2S BMS. If unsure, buy a prebuilt 2S pack or get help.
  2. Wire pack -> buck converter. Adjust the buck to exactly 5.00 V with a multimeter before connecting the Pi.
  3. Wire motor driver Vcc to the buck (or separate buck for motors), connect motor outputs to wheels. Add a 2200–4700 µF electrolytic across the motor 5 V rail.
  4. Mount Pi safely, insert microSD with OS. Mount camera on the forward mast and connect ribbon cable.

Step 3 — Software (1–2 hrs)

  1. Flash Raspberry Pi OS (Lite or Desktop) and install Python packages: opencv-python, pyzbar, pytesseract (if using OCR), gpiozero or pigpio.
  2. Clone the repo or copy scripts: camera capture → decode → lookup → motion routine. Add a small Flask web UI for manual control and teaching poses if desired.
  3. Write a small "teach" tool: jog servos via web UI, save pose CSV line (id,name,time,angles) to local DB (CSV/JSON/SQLite).

Step 4 — Calibration & testing (45–90 mins)

  1. With MediBot powered but motors stopped, jog servos and save 3–5 demo poses for common medicine slots.
  2. Test camera decode at expected sticker sizes and distances. Tune lighting and contrast if needed.
  3. Run the full dispense routine once per med to verify grip strength and travel path. Adjust servo angles if the bottle slips.
  4. Test scheduled run and logging. Add retry logic for failed grabs.

Maintenance & upgrades

  • Replace SG90 servos with metal‑geared higher torque types if you carry heavier bottles.
  • Swap L293D for TB6612/MOSFET driver to reduce heat and increase motor voltage to the wheels.
  • Add small UV/cleaning cup or disposable liners to avoid direct contact with medicine surfaces.
